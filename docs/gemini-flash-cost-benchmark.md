## 분석

### 가격 구조 분석

```
입력 단가 비율 = $1.50 / $3.00 = 50.00%  ← 정확히 50%
출력 단가 비율 = $9.00 / $15.00 = 60.00%  ← 출력이 많으면 불리
```

**핵심 인사이트**: Gemini Flash의 비용 우위는 **입력 토큰 비중**에 비례합니다.

- 입력:출력 ≥ 100:1 → Gemini/Claude ≈ **50.0%** (50% 기준 충족)
- 입력:출력 = 20:1 → Gemini/Claude ≈ **50.5%** (기준 근접)
- 입력:출력 < 5:1 → Gemini/Claude > **52%** (기준 초과)

### Jarvis 적용 적합성

Jarvis의 대용량 문서 처리 태스크는 **입력 집약적** 특성을 가집니다:
- 뉴스 요약: 문서 10K~100K 토큰 입력, 500 토큰 출력 → 입출력비 20~200:1
- RAG 전처리: 문서 100K+ 토큰 입력, 1K 토큰 출력 → 입출력비 100:1+
- 문서 분석: 문서 50K~1M 토큰 입력, 2K 토큰 출력 → 입출력비 25~500:1

**결론: Jarvis 대용량 문서 처리 태스크에서 Gemini Flash는 Claude Sonnet 대비 약 50% 비용으로 동일 작업 가능.**
