# Jarvis — Codebase Structure

## Top Layout

```
~/jarvis/
├── CLAUDE.md            # Project instructions (loaded by Claude Code session start)
├── README.md / README.ko.md
├── infra/               # Main code area
├── rag/                 # @jarvis/rag npm package (LanceDB)
├── docs/                # Images, promo
├── scripts/             # Setup wizards
└── private/             # Sensitive helpers
```

## infra/ — Main automation surface

```
infra/
├── adr/                 # Architecture Decision Records
├── agents/              # Agent prompts/configs
├── bin/                 # Executable entrypoints (most-touched)
│   ├── ask-claude.sh    # CORE — LLM call wrapper
│   ├── bot-cron.sh      # Task dispatcher (reads tasks.json → script or retry-wrapper)
│   ├── jarvis-cron.sh   # Same as bot-cron variant
│   ├── retry-wrapper.sh # Retry policy + error classification
│   ├── plugin-loader.sh # Regenerates effective-tasks.json from tasks.json + plugins/
│   └── jarvis-init.sh   # First-time setup
├── chrome-extension/    # MCP Chrome extension (browser automation)
├── config/              # tasks.json, monitoring.json, empty-mcp.json
├── context/             # Per-task context files (loaded into prompts)
├── discord/             # Discord bot (Node.js)
│   └── lib/handlers.js  # Command/message handlers
├── docs/                # OPERATIONS.md, INDEX.md
├── lib/                 # Shared shell + helper libraries
│   ├── llm-gateway.sh   # Multi-provider LLM router (Claude → OpenAI → Ollama)
│   ├── evaluator.sh     # Tier 1 result quality gate (NEW)
│   ├── insight-recorder.sh # Auto-extract insights to vault
│   ├── coder-functions.sh # Shared coder helpers
│   ├── log-utils.sh     # Structured logging
│   ├── semaphore.sh     # Concurrency control (mkdir-based)
│   ├── event-bus.sh     # Event-driven hook system
│   └── stream-to-board.sh # Stream LLM activity to board
├── plugins/             # File-convention plugins (ADR-007)
├── prompts/             # External prompt files (.md)
├── scripts/             # Helper scripts (largest dir)
│   ├── token-ledger-audit.sh # Weekly self-audit (Tier 0)
│   ├── tune-task-params.sh   # Tier 5 KPI auto-tune suggestions
│   ├── github-monitor-gate.sh # Hash cache gate example
│   ├── gate-factory/    # Reusable gate pattern docs
│   └── task-executors/  # Tier 3 plan/execute/verify wrappers
└── package.json         # puppeteer-core only
```

## rag/ — RAG module (separate npm package)

```
rag/
├── bin/                 # rag-index, rag-stats, rag-compact, rag-repair
├── lib/                 # RAGEngine, paths, query helpers
└── package.json         # @lancedb/lancedb, apache-arrow, chokidar
```

## State directory (~/.jarvis/)
- `config/tasks.json` — SSoT task definitions
- `config/effective-tasks.json` — generated by plugin-loader.sh
- `state/token-ledger.jsonl` — Tier 0 LLM cost ledger (NEW)
- `state/circuit-breaker/*.json` — failure tracking
- `state/gate-candidates.json` — Tier 2 dedup candidate queue (NEW)
- `state/context-bus.md` — shared context between agents
- `state/board-minutes/*.md` — daily board meeting outputs
- `logs/cron.log` — primary execution log
- `logs/task-runner.jsonl` — per-task structured log
- `logs/claude-stderr-<task>-<date>.log` — per-task stderr (rotated 14d)
- `results/<task>/<datetime>.md` — per-task LLM outputs (rotated by retention)
