# Jarvis — Project Overview

## Purpose
**AI operations platform that manages itself 24/7**. Discord Bot + RAG Knowledge Base + Insight Layer + Self-Healing Automation. Runs on Claude Max subscription with 100% local data ($0 API charges). Personal-use AI ops for owner Jeongwoo.

## Tech Stack
- **Language**: Bash (primary), Node.js 22+ (Discord bot, RAG), Python 3.12 (utilities, voice), TypeScript (jarvis-board separate repo)
- **LLM**: `claude` CLI (Claude Code) wrapped via `infra/bin/ask-claude.sh`. Optional Ollama llama3.2 fallback for voice agent.
- **RAG**: LanceDB vector store + Ollama embeddings (hybrid search). Located in `rag/` subdir as `@jarvis/rag` package.
- **Discord**: Node.js bot in `infra/discord/`, slash commands + DM handling.
- **Scheduler**: macOS launchd + crontab. Tasks defined in `~/.jarvis/config/tasks.json` (SSoT) → `effective-tasks.json` regenerated by `plugin-loader.sh`.

## Repos
- `~/jarvis/` — code repo (this project)
- `~/.jarvis/` — runtime state directory (logs, config, results, RAG data, sessions)
- `~/jarvis-board/` — separate Next.js virtual office map (different repo)

## Key Architecture Layers
| Layer | Component | Path |
|---|---|---|
| Wrapper | `ask-claude.sh` (LLM call core) | `infra/bin/ask-claude.sh` |
| Dispatcher | `bot-cron.sh` (task router) | `infra/bin/bot-cron.sh` |
| Retry | `retry-wrapper.sh` (failure classification) | `infra/bin/retry-wrapper.sh` |
| Gateway | `llm-gateway.sh` (multi-provider router) | `infra/lib/llm-gateway.sh` |
| Evaluator | `evaluator.sh` (Tier 1 quality gate) | `infra/lib/evaluator.sh` |
| Token Ledger | append-only JSONL | `~/.jarvis/state/token-ledger.jsonl` |
| Weekly Audit | `token-ledger-audit.sh` | `infra/scripts/token-ledger-audit.sh` |
