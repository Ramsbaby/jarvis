## 1. 영상 핵심 방법론

### 세컨브레인 정의
> "모든 지식의 집합체 ... AI 에이전트가 잘 액세스할 수 있도록 만들어진 그릇/저장소" — 아이언맨 자비스 지향.

동기: **사람이 병목(바틀넥)** — "내가 없어도 에이전트가 나의 맥락을 이해해 스스로 돌아가게."

### 3층 구조 (typed graph 패턴)
1. **원자 단위 마크다운 파일** — 생각을 잘게 쪼개 텍스트로 저장
2. **타입이 있는 그래프** — 노드(통찰/절차/사건/주장) + 엣지(지지/확장/유사/반박)
3. **에이전트(클로드 코드)가 그래프를 스스로 생성** — 소스 정제 + 기존 지식과 연결

원형: 안드레이 카파시의 "LLM 위키" 패턴 (유튜브 스크립트/슬라이드를 소스로, LLM이 유지하는 상호링크 지식 그래프 + 옵시디언 볼트).

### 5대 원칙
1. **맥락이 유일한 차별점** — "지식은 공짜, 진짜 무기는 나만 가진 맥락. 무엇을 먹여서 시키느냐가 전부."
2. **RAG가 아니라 컴파운딩(복리)** — "질문마다 새로 검색이 아니라, 소스를 위키 연구로 반영해 성장. 내 위키가 곧 성장."
3. **옵시디언(UI)에서 멈추지 말 것** — "실제 활용은 클로드 코드로 위키에 쿼리. UI+에이전트 결합."
4. **출처 표기 필수** — 할루시네이션 방지, 검증 가능성.
5. **휴먼 터치 유지** — 자동화 100% 의존 금지. "라이브는 카피 불가."

### 함정 (안티패턴)
- "3만 개 검색되면 없는 거랑 똑같다" — 규모 폭증 시 검색 무용화 → 캐싱/인덱스 층 필요
- 키워드 검색 의존 — 정보는 결정론적이지 않음 → LLM/임베딩 필요
- 옵시디언이 끝이라는 착각 — 대부분 여기서 멈춤
- UI에만 기대기 — 사람이 못 봄 → UI+에이전트
- "완벽한 뇌 구조" 정답 찾기 — 단일 정답 스키마 없음

7. **컴파운딩 인사이트 저장소** — eureka.jsonl
