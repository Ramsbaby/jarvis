## 메타 통찰
**자비스는 영상을 따라할 학생이 아니라, 영상이 비추는 거울 앞의 고수다.** 영상의 99%를 이미 구현(7-Gate 검증·Sprint Contract·멀티에이전트·오답노트 3,316건). 그래서 영상은 교본이 아니라 거울이며, 비추는 건 **자비스의 강점(하네스 축적)이 그대로 약점이 된 "축적의 함정"**이다. 두 영상의 교집합 메시지: **"양이 아니라 배치와 강제다."**

## 3대 전략 (우선순위순)

### 전략 1 (최우선): 룰 다이어트 + 강제 전환 — "0% 커버리지" 해소
- **근거**: 2,266룰 중 코드 강제 3개. 나머지는 "프롬프트에 적고 LLM이 읽길 바라는" 텍스트. 영상1 경고 직격.
- **실행**:
  - (a) 텍스트 룰 → 강제 hook 승격. 예: 오늘 추가한 "jarvis-boram 발송 금지"를 _capabilities.md 텍스트로 두지 말고 discord 발송 함수 채널 화이트리스트 가드로.
  - (b) BLOCKING 57건 ROI 감사: 죽은 룰 아카이브 + 유사 룰 병합 + 핵심 7개만 STABLE 상위 고정.
  - (c) rule-hook-coverage 감사 재가동(3주 정지) + mistake-promoter에 "추가보다 hook 승격 우선" 방향 가드.
- **효과**: 프롬프트 슬림화 → 핵심 룰이 묻히지 않음 → LLM 준수율↑.

### 전략 2: 능동 결정 원장 — 영상2 ④ "비서" 승격
- **근거**: 메모리가 자동추출 중심. 오늘 세션만 봐도 진행중 결정 5건(토큰감시 18시 검증·사람인 승인대기·job-scout 12소스·jarvis-boram 가드·전략3건)이 다음 세션에 명시 인수인계 안 됨.
- **실행**: 결정/대기 원장(decision ledger) 신설 + SessionStart가 "지난 결정·대기 먼저 브리핑".

### 전략 3: 브라우저 자가검증 강제 하네스 — 영상1 ② 유일 갭
- **근거**: Java는 자동 컴파일·실행 검증인데 웹UI(자비스맵)는 강제 하네스 없음. CDP 로컬크롬 패턴은 job-scout에서 이미 실측 검증됨 → 재활용.
- **실행**: 자비스맵 수정 시 Playwright/CDP 자동 스모크 → 허위 완료 차단(Iron Law 6 도구화).
