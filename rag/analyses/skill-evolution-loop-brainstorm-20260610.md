# 스킬 자가 생성 루프 — /brainstorm 산출물 (2026-06-10)

> ADR-021(자가 진화 루프) 활성화 설계. 헤르메스(NousResearch hermes-agent) 패턴의 자비스 이식.
> 12개 역질문 전체에 주인님 답변 완료. 다음 단계: /autoplan.

## 확정 결정 12개 (주인님 결재 완료)

1. **목적**: 내부 능력 복리 성장 + OSS 차별 기능 **둘 다 동등** — 내부 루프 먼저 완성, 검증 후 OSS 플러그인으로 분리 공개 (2단계)
2. **트리거**: **야간 크론 배치** (하루치 일괄 분석, 비용 1회 통제)
3. **결재 게이트**: **초안 생성 → 주인님 결재 후 등재** (drafts/ 격리 → 승인 시 skills/ 승격)
4. **성공 지표**: **승인율(1차 게이트) + 승격 스킬 재사용 횟수(최종)** — 둘 다 ledger 기록
5. **성공 세션 판정**: **휴리스틱 프리필터 + LLM 스코어링 2단** (비LLM 필터로 80%+ 컷 → 통과분만 LLM 판정)
6. **최악 시나리오(가드 우선순위)**: ① 쓰레기 초안 양산 ② PII·시크릿 유출 (복수 선택)
7. **입력 소스**: **기존 요약 체인 재사용** (stop-wiki-ingest·session-summarizer·learn JSONL — 신규 수집 0, DRY)
8. **타겟 표면**: **자비스 생태계 전체** — Claude Code 스킬 + 디스코드 봇 스킬 + 크론 태스크 (공통 frontmatter `target:` 필드로 표현)
9. **결재 UX**: **Discord 카드+버튼 즉시 알림 + 월요일 묶음 재안내 병행**
10. **생성 상한**: **일 2~3건 + 점수 임계치** (둘 다 통과해야 초안화)
11. **skill-creator 관계**: **생성 엔진으로 재사용** — 배치는 후보 선별+재료 준비까지, 초안 작성은 skill-creator 포맷·프롬프트 호출
12. **헤르메스 대비 차별**: **검증 증거 의무 + 오답노트 연계** — 실측 명령·출력 없는 절차는 초안 미생성(환각 절차 차단), 관련 오답노트 가드를 스킬 본문에 자동 포함 (실패·성공 양방향 학습)

## 암묵적 가정 12개 (구현 전 확인 필요)

- [A1] 기존 세션 요약에 절차 디테일이 충분히 남아 있다 | 깨지면: 뜬구름 초안→승인율 폭락 | 확인: 최근 요약 3개로 초안 1건 수동 생성 테스트
- [A2] 일일 세션 수가 야간 배치 1회로 처리 가능한 규모다 | 깨지면: 배치 시간 폭주 | 확인: 세션 로그 일일 건수 실측
- [A3] 휴리스틱 프리필터로 80%+ 컷 가능 | 깨지면: LLM 판정 비용 증가 | 확인: DRYRUN 1주 분포 실측
- [A4] 3개 표면(CLI/디스코드/크론)을 공통 초안 포맷 1종으로 표현 가능 | 깨지면: 포맷 3종 분기→복잡도 3배 | 확인: frontmatter `target: cli|discord|cron` 설계 검증
- [A5] skill-creator가 비대화식(배치, claude -p) 호출로 동작한다 | 깨지면: 독립 생성기로 회귀 | 확인: 비대화 모드 1회 시험
- [A6] Discord 버튼 결재→파일 등재 권한 체인이 안전하다 | 깨지면: 봇이 ~/.claude/skills/ 쓰기 권한 보유 = 공격면 | 확인: 등재 실행을 봇이 아닌 크론/CLI 측에 두는 분리 설계
- [A7] 기존 privacy guard regex 재사용으로 스크럽 충분 | 깨지면: 사내 도메인 지식 유출 | 확인: 회사명·사내 시스템명(메타에이전트·에피소리·핵토 등) blocklist 추가
- [A8] 검증 증거는 세션 로그의 실제 명령 출력에서 추출 가능 | 깨지면: 환각 절차 스킬화 | 확인: 증거 0건 → 초안 미생성 하드 게이트
- [A9] 일 2~3건이면 결재 피로 지속 가능 | 깨지면: 미결재 적체→폴더 방치 | 확인: 14일 미결재 자동 만료(아카이브) 정책 병행
- [A10] 기존 스킬 ~40종과의 중복을 유사도 검사로 감지 가능 | 깨지면: 아류 스킬→트리거 혼선 | 확인: 초안 생성 전 기존 스킬 카탈로그 주입 + "중복이면 기존 스킬 개선 제안으로 전환" 룰
- [A11] 지표 측정은 기존 ledger 패턴 복제로 가능 | 깨지면: 측정 불가→ADR-021 백로그 회귀 | 확인: skill-loop ledger JSONL 1개 신설 (생성·결재·호출 3이벤트)
- [A12] 신규 크론 1개가 CRON-INTRODUCTION-CHECKLIST 통과 가능 | 깨지면: 도입 반려 | 확인: Why 1줄 = "성공 절차가 휘발되어 같은 작업을 매번 처음부터 다시 하는 낭비 차단", DRYRUN 1주 포함

## Discord 결재 카드 — 방향 3안 (데이터 앱 카테고리: 다크/차분한 회색, AI slop 금지)

- **방향 A — 한 줄 결재 (미니멀)**: 스킬명·한 줄 요약·재사용 가치 점수·증거 개수만 + [승인/보류/폐기] 버튼. 임베드 컬러 슬레이트(#44545B). Galaxy에서 3초 결재. 상세는 접힌 링크.
- **방향 B — 근거 우선 (감사형)**: 검증 증거 발췌(명령+출력 2줄)와 출처 세션 링크를 카드 전면 배치. 결재 10초 걸리지만 환각 차단 최강. 컬러 차분한 인디고.
- **방향 C — 주간 다이제스트 (묶음형)**: 개별 카드 없이 월요일 1장 요약 + 번호 응답 결재. 알림 피로 0, 즉시성 포기.
- **권고**: 결재 UX "둘 다" 답변에 따라 **A(즉시) + C(월요일 묶음) 조합, B의 증거 발췌 1줄을 A 카드에 삽입**.

## 놓친 부분 3개 점검

1. **접근성**: 카드 본문 bullet 고정 (Discord 모바일 표 깨짐 — 기존 시각화 정책 준수)
2. **장애 시나리오**: 배치 실패(토큰 만료·Ollama 다운) 시 기존 크론 실패 알림 체인 재사용, 당일 스킵·익일 누적 재분석 금지(폭주 방지)
3. **관측·롤백**: 자동 생성 스킬에 `origin: auto-generated` 태그 의무 → 주간 감사에서 미사용·문제 유발 스킬 자동 보고, 롤백은 drafts/archive로 비파괴 이동

## 요구사항 명세 (10줄)

1. 성공 세션의 절차를 재사용 스킬로 복리 축적한다 — 내부 우선, 검증 후 OSS 분리
2. 야간 크론 1개가 기존 세션 요약 체인을 입력으로 후보를 선별한다 (신규 수집 0)
3. 선별은 휴리스틱→LLM 2단, 점수 임계치 + 일 2~3건 상한
4. 초안 작성은 skill-creator를 엔진으로 호출, frontmatter `target:`으로 3표면 커버
5. 실측 검증 증거 섹션 의무 — 증거 0건이면 초안 자체를 만들지 않는다
6. 관련 오답노트 가드를 스킬 본문에 자동 포함한다 (실패·성공 양방향 학습)
7. PII·시크릿·사내명 스크럽 + 기존 스킬 유사도 검사 (중복 시 개선 제안 전환)
8. drafts/ 격리 → Discord 카드(A+C 조합) 결재 → 승인 시 등재, 14일 미결재 자동 만료
9. skill-loop ledger로 승인율·재사용 횟수 측정, 주간 감사 연동, 비파괴 롤백
10. 도입은 CRON 체크리스트 준수 — DRYRUN 1주(초안만, 알림 X) 후 본 가동

## 가정 실측 결과 (2026-06-10 13:45 KST — autoplan 진입 전 완료)

- **[A1] 부분 통과 → 설계 보정**: 요약 체인(위키 facts·discord-history)은 "사실 한 줄" 위주라 **선별용으로만 충분**, 절차·검증 증거 추출은 불가.
