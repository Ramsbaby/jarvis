## 2. Jarvis 월간 비용 영향 분석

### 2.1 현재 구성 (Claude Sonnet 기준)

**가정**:
- 월간 처리량: 500M tokens
  - 평균 100K 입력, 1K 출력 당 1건 작업
  - 월간 약 5,000건 작업 (크론 태스크 + 사용자 요청)

**계산**:

```
월간 입력: 500M tokens
월간 출력: 5M tokens
(가정: 입출력 비율 1:10)

Claude Sonnet 비용:
= (500M × $3/1M) + (5M × $15/1M)
= $1,500 + $75
= $1,575/월
```

### 2.2 Gemini Flash 도입 후

**라우팅 전략**:
- **Claude** (고비용, 고성능): 복잡한 논리·의사결정 작업 (20%)
- **Gemini Flash** (중비용, 1M 컨텍스트): 대용량 문서 처리 (50%)
- **DeepSeek** (초저가): 요약·분류 (30%)

**계산**:

```
Claude 비용 (20% 할당):
= (100M × $3/1M) + (1M × $15/1M)
= $300 + $15
= $315

Gemini Flash 비용 (50% 할당):
= (250M × $1.50/1M) + (2.5M × $9/1M)
= $375 + $22.50
= $397.50

DeepSeek 비용 (30% 할당):
= (150M × $0.14/1M) + (1.5M × $0.28/1M)
= $21 + $0.42
= $21.42

총 비용 = $315 + $397.50 + $21.42 = $733.92/월
```

### 2.3 비용 절감 효과

```
현재 (Claude만): $1,575/월
제안 (멀티모델): $734/월

절감액: $841/월 (53.4% 절감)
연간절감: $10,092
```

## 3. Gemini Flash의 핵심 강점

### 3.1 기술적 장점

1. **1M 토큰 컨텍스트 윈도우**
   - Claude Sonnet: 200K (일부), Opus: 200K
   - Gemini Flash: 1M (풀 지원)
   - 매우 긴 문서를 단일 호출로 처리 가능

2. **비용 효율**
   - 입력 토큰: $1.50/1M (Sonnet의 50%)
   - 대용량 문서 처리에 최적화

3. **응답 속도**
   - Gemini Flash는 경량 모델로 응답 시간 단축
   - 평균 처리 시간: 5-15초 (문서 크기별)

4. **캐시 지원 예정**
   - Gemini Batch API (향후): 추가 50% 할인
   - 반복적인 작업에 추가 절감 가능
